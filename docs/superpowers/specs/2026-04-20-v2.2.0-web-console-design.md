## 1. 概述

### 1.1 目标

为ClaudeFlow提供Web可视化控制台，实现：
- 任务列表展示与状态过滤
- 7阶段工作流可视化（Phase0-6）
- 当前阶段步骤进度展示
- 任务ID复制与恢复（✅ 已修正）
- 等待介入任务高亮与操作

### 1.2 技术栈

| 层 | 技术 |
|----|------|
| 前端 | Vue 3 + Vite + TypeScript |
| 后端 | Spring Boot 3.x 单模块架构 |
| 存储 | SQLite（`cf01_`表前缀） |
| 通信 | Python→Java WebSocket, Java→Vue SSE |
| 端口 | 可配置，默认21000 |


### 3.2 任务ID区域

> **⚠️ 2026-04-21 修正**：原设计误用了Claude Code CLI的session_id概念，已修正为Hermes任务ID。
>
> **修正原因**：用户向Hermes发布指令，不直接操作Claude Code CLI。Claude Code CLI的session_id是Hermes内部管理的实现细节，不应暴露给用户。

**任务ID获取方式（Hermes CLI）**：

```bash
# 方式1：任务创建时返回
hermes create "实现Web控制台模块"
# 输出：任务已创建，ID: task_001abc23

# 方式2：进入Hermes后获取（如用户所述）
hermes  # 进入Hermes交互模式
# 此时Hermes显示当前session_id/task_id
# 用户可直接记录或退出后使用 hermes resume task_xxx 恢复

# 方式3：查看任务列表
hermes list
# 输出所有任务及其ID
```

**UI设计**：
- 浅蓝色背景区域 `#e3f2fd` + 边框 `#2196F3`
- 任务ID显示：`font-family: monospace`
- 一键复制按钮
- 使用提示：`hermes resume task_xxx` 或 Web控制台"恢复任务"按钮

```
┌─────────────────────────────────────────────────────────┐
│ 🔗 Hermes任务                                            │
│ 任务ID: task_001abc23    [📋 复制任务ID]                │
│ 💡 复制后可用 hermes resume task_001abc23 继续          │
│ 或在Web控制台点击"恢复任务"按钮                          │
└─────────────────────────────────────────────────────────┘
```

**架构说明**（内部实现，用户不感知）：

```
┌─────────────────────────────────────────────────────────────┐
│  用户交互层                                                  │
│  用户 → Hermes Web控制台 / Hermes CLI                       │
│  用户向 Hermes 发布指令（task_id 是用户操作的唯一标识）       │
└─────────────────────────────────────────────────────────────┘
                          ↓
┌─────────────────────────────────────────────────────────────┐
│  Hermes调度层                                                │
│  Hermes → 驱动 Claude Code CLI 执行子任务                    │
│  Hermes内部管理多个CLI进程，Claude Code session_id 是内部细节 │
└─────────────────────────────────────────────────────────────┘
                          ↓
┌─────────────────────────────────────────────────────────────┐
│  Claude Code CLI执行层（用户不可见）                         │
│  Claude Code CLI → 执行具体代码操作                          │
│  CLI的session_id由Hermes内部管理，不暴露给用户               │
└─────────────────────────────────────────────────────────────┘
```

原设计中的`session_id`字段改为内部管理字段，不作为用户交互标识：

## 4. 后端API设计

### 4.1 表结构

```sql
CREATE TABLE cf01_task (
    id VARCHAR(36) NOT NULL COMMENT '任务ID',
    name VARCHAR(100) NOT NULL COMMENT '任务名称',
    domain VARCHAR(50) NOT NULL COMMENT '所属领域',
    status VARCHAR(20) NOT NULL COMMENT '状态',
    phase VARCHAR(50) DEFAULT '' COMMENT '当前阶段',
    progress INTEGER DEFAULT 0 COMMENT '进度百分比',
    session_id VARCHAR(100) DEFAULT NULL COMMENT 'CLI会话ID（待验证项）',
    description TEXT COMMENT '任务描述',
    priority VARCHAR(10) DEFAULT '中' COMMENT '优先级',
    gmt_create DATETIME DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
    gmt_modified DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '修改时间',
    PRIMARY KEY (id)
);

CREATE TABLE cf01_checkpoint (
    id VARCHAR(36) NOT NULL COMMENT 'CheckpointID',
    task_id VARCHAR(36) NOT NULL COMMENT '任务ID',
    phase VARCHAR(50) NOT NULL COMMENT '阶段',
    step_index INTEGER DEFAULT 0 COMMENT '步骤索引',
    summary TEXT COMMENT '阶段总结',
    internal_cli_session_id VARCHAR(100) DEFAULT NULL COMMENT '内部CLI会话ID（Hermes内部管理，用户不可见）',
    gmt_create DATETIME DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
    PRIMARY KEY (id),
    FOREIGN KEY (task_id) REFERENCES cf01_task(id)
);

CREATE TABLE cf01_phase_step (
    id VARCHAR(36) NOT NULL COMMENT '步骤ID',
    task_id VARCHAR(36) NOT NULL COMMENT '任务ID',
    phase VARCHAR(50) NOT NULL COMMENT '阶段',
    step_index INTEGER NOT NULL COMMENT '步骤索引',
    step_name VARCHAR(100) COMMENT '步骤名称',
    status VARCHAR(20) DEFAULT 'pending' COMMENT '步骤状态',
    gmt_create DATETIME DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
    gmt_modified DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '修改时间',
    PRIMARY KEY (id),
    FOREIGN KEY (task_id) REFERENCES cf01_task(id)
);
```

### 实现要点

#### 1. Web后端启动Hermes Worker

```java
// TaskController.java
@PostMapping("/api/tasks")
public Task createTask(@RequestBody TaskRequest request) {
    // 1. 创建任务记录
    Task task = taskService.createTask(request.getDescription());
    
    // 2. 启动Hermes Worker进程
    ProcessBuilder pb = new ProcessBuilder(
        "python", "hermes/main.py",
        "--task-id", task.getId(),
        "--mode", "worker"
    );
    pb.directory(new File("/path/to/claudeflow"));
    Process worker = pb.start();
    
    // 3. 记录Worker进程ID（用于后续管理）
    task.setWorkerPid(worker.pid());
    taskService.updateTask(task);
    
    // 4. 返回任务ID给前端
    return task;
}
```

#### 3. 恢复中断任务

```java
// TaskController.java
@PostMapping("/api/tasks/{taskId}/resume")
public Task resumeTask(@PathVariable String taskId) {
    Task task = taskService.getTask(taskId);
    
    // 检查是否有中断的Worker进程
    if (task.getWorkerPid() != null && !isProcessAlive(task.getWorkerPid())) {
        // 重新启动Worker进程
        ProcessBuilder pb = new ProcessBuilder(
            "python", "hermes/main.py",
            "--task-id", taskId,
            "--mode", "resume",
            "--checkpoint", task.getLastCheckpointId()
        );
        Process worker = pb.start();
        task.setWorkerPid(worker.pid());
    }
    
    task.setStatus("running");
    taskService.updateTask(task);
    
    // WebSocket通知Worker继续执行
    webSocketHandler.sendToWorker(taskId, "{\"type\":\"task_command\",\"command\":\"resume\"}");
    
    return task;
}
```
