# V2.2.0 Web控制台设计文档

> **日期**: 2026-04-20
> **版本**: V2.2.0
> **范围**: Dashboard页面 + TaskDetail页面 + 后端API

---

## 1. 概述

### 1.1 目标

为ClaudeFlow提供Web可视化控制台，实现：
- 任务列表展示与状态过滤
- 7阶段工作流可视化（Phase0-6）
- 当前阶段步骤进度展示
- 会话ID复制（待验证项）
- 等待介入任务高亮与操作

### 1.2 技术栈

| 层 | 技术 |
|----|------|
| 前端 | Vue 3 + Vite + TypeScript |
| 后端 | Spring Boot 3.x 单模块架构 |
| 存储 | SQLite（`cf01_`表前缀） |
| 通信 | Python→Java WebSocket, Java→Vue SSE |
| 端口 | 可配置，默认21000 |

## 2. Dashboard页面设计

### 2.1 页面布局

```
┌─────────────────────────────────────────────────────────────┐
│ 顶部统计区（4卡片）                                          │
│ ┌─────────┐ ┌─────────┐ ┌─────────┐ ┌─────────┐            │
│ │ 运行中  │ │ 已完成  │ │等待介入 │ │  告警   │            │
│ │   3     │ │   12    │ │    1    │ │    0    │            │
│ └─────────┘ └─────────┘ └─────────┘ └─────────┘            │
├─────────────────────────────────────────────────────────────┤
│ 任务列表区                                                   │
│ ┌───────────────────────────────────────────────────────┐  │
│ │ task_001 │ 运行中 │ Phase0-6工作流 │ [展开详情]       │  │
│ │ task_002 │ 已完成 │ Phase0-6工作流 │                  │  │

│ └───────────────────────────────────────────────────────┘  │
└─────────────────────────────────────────────────────────────┘
```

### 2.2 统计卡片交互

- **点击卡片** → 任务列表联动过滤对应状态
- **卡片样式**：
  - 运行中：绿色背景 `#4a4`
  - 已完成：灰色背景 `#ddd`
  - 等待介入：橙色背景 `#ff9800`，高亮显示
  - 告警：红色背景 `#c00`

### 2.3 任务卡片工作流展示

**7阶段工作流可视化**：

```
Phase0 ── Phase1 ── Phase2 ── Phase3 ── Phase4 ── Phase5 ── Phase6
  ●        ●        ●        ◉        ○        ○        ○
拆分     规划     准备     开发     审查     测试     完成
完成     完成     完成    进行中    待执行    待执行    待执行
```

**节点状态样式**：
- 已完成：实心圆 + 绿色 `#4a4` + "✓ 已完成"
- 进行中：实心圆 + 蓝色 `#2196F3` + 发光阴影 + "进行中 N/M"
- 待执行：空心圆 + 灰色 `#ddd` + "待执行"

**节点可点击**：点击Phase节点展开/收起该阶段的步骤详情。

### 2.4 当前阶段步骤展示

**交互方式**：
- 横向滚动：步骤卡片横向排列
- 鼠标拖拽：支持鼠标左右拖拽滚动
- 小手光标：
  - 默认状态 `cursor: grab`
  - 拖拽中 `cursor: grabbing`
- 触摸支持：移动端支持触摸滑动
- 滚动提示：显示 "◀ 左右滑动查看更多 ▶"

**步骤状态样式**：
- 已完成：绿色背景 + "✓ 步骤N"
- 进行中：蓝色背景 + 发光 + "▶ 步骤N"
- 待执行：灰色背景 + "○ 步骤N"

### 2.5 等待介入任务处理

- **高亮显示**：橙色边框 + 闪烁动画
- **操作按钮**：
  - "提供信息" → 弹窗输入补充信息
  - "取消任务" → 确认后取消

### 3.2 会话ID区域

**⚠️ 待验证项**：

会话ID的获取方式需要验证：
- 方案A：CLI主动传递（推荐）
  - CLI在WebSocket消息中添加`session_id`字段
  - 需验证CLI是否能获取当前会话ID
- 方案B：Python自动关联
  - ClaudeFlow启动时记录会话ID
  - 可能存在会话重启后关联断裂问题

**当前状态**：
- Task数据模型已预留`session_id`字段
- 后端API已设计接收session_id参数
- 具体获取途径待CLI环境验证

**UI设计**：
- 橙色背景区域 `#fff3e0` + 边框 `#ff9800`
- 会话ID显示：`font-family: monospace`
- 一键复制按钮
- 使用提示：`claude --resume session_xxx`

### 3.3 Checkpoint历史

## 4. 后端API设计

### 4.1 表结构

```sql
CREATE TABLE cf01_task (
    id VARCHAR(36) NOT NULL COMMENT '任务ID',
    name VARCHAR(100) NOT NULL COMMENT '任务名称',
    domain VARCHAR(50) NOT NULL COMMENT '所属领域',
    status VARCHAR(20) NOT NULL COMMENT '状态',
    phase VARCHAR(50) DEFAULT '' COMMENT '当前阶段',
    progress INTEGER DEFAULT 0 COMMENT '进度百分比',
    session_id VARCHAR(100) DEFAULT NULL COMMENT 'CLI会话ID（待验证项）',
    description TEXT COMMENT '任务描述',
    priority VARCHAR(10) DEFAULT '中' COMMENT '优先级',
    gmt_create DATETIME DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
    gmt_modified DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '修改时间',
    PRIMARY KEY (id)
);

CREATE TABLE cf01_checkpoint (
    id VARCHAR(36) NOT NULL COMMENT 'CheckpointID',
    task_id VARCHAR(36) NOT NULL COMMENT '任务ID',
    phase VARCHAR(50) NOT NULL COMMENT '阶段',
    step_index INTEGER DEFAULT 0 COMMENT '步骤索引',
    summary TEXT COMMENT '阶段总结',
    gmt_create DATETIME DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
    PRIMARY KEY (id),
    FOREIGN KEY (task_id) REFERENCES cf01_task(id)
);

CREATE TABLE cf01_phase_step (
    id VARCHAR(36) NOT NULL COMMENT '步骤ID',
    task_id VARCHAR(36) NOT NULL COMMENT '任务ID',
    phase VARCHAR(50) NOT NULL COMMENT '阶段',
    step_index INTEGER NOT NULL COMMENT '步骤索引',
    step_name VARCHAR(100) COMMENT '步骤名称',
    status VARCHAR(20) DEFAULT 'pending' COMMENT '步骤状态',
    gmt_create DATETIME DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
    gmt_modified DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '修改时间',
    PRIMARY KEY (id),
    FOREIGN KEY (task_id) REFERENCES cf01_task(id)
);
```

### 4.2 API端点

| 端点 | 方法 | 说明 |
|------|------|------|
| `/api/tasks` | GET | 任务列表（支持status过滤） |
| `/api/tasks/stats` | GET | 统计数据（运行中/已完成/等待介入/告警） |
| `/api/tasks/{id}` | GET | 任务详情 |
| `/api/tasks/{id}/pause` | POST | 暂停任务 |
| `/api/tasks/{id}/resume` | POST | 恢复任务 |
| `/api/tasks/confirm/{id}` | POST | 确认等待介入任务（提交补充信息） |
| `/api/tasks/cancel/{id}` | POST | 取消任务 |
| `/api/checkpoints/{taskId}` | GET | Checkpoint历史 |
| `/api/checkpoints/revert/{checkpointId}` | POST | 回退到Checkpoint |
| `/api/steps/{taskId}/{phase}` | GET | 阶段步骤列表 |
| `/api/events/stream` | GET | SSE实时事件流 |

## 6. 数据清理策略

### 6.1 清理规则

- **保留周期**：7天
- **清理时间**：每日03:00执行
- **清理范围**：
  - 已完成/已取消任务
  - 对应的Checkpoint记录
  - 对应的PhaseStep记录

### 6.2 定时任务配置

```java
@Scheduled(cron = "0 0 3 * * ?")
public void cleanupOldTasks() {
    LocalDateTime threshold = LocalDateTime.now().minusDays(7);
    taskRepository.deleteByStatusAndGmtModifiedBefore(
        Arrays.asList("completed", "archived"),
        threshold
    );
}
```
